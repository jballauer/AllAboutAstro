# Gallery AI-facts stub

A violet, dashed-border disclosure box (`.ke-ai-stub`, defined once in
`BaseLayout.astro`) marking a paragraph of facts as AI-compiled research
about the astronomical subject itself — not Jay's own notes, observing
log, or first-hand description. Every Astro Gallery entry gets one,
regardless of whether Jay also wrote real first-hand content on the same
page (per `feedback_ke_stub_placeholder_content` memory's 2026-07-23
extension: this box stopped being "only for pages with nothing else" —
it now runs alongside full Jay-written narratives too, e.g. the M45
Mosaic and Great American Eclipse 2017 pages).

**Scope: astronomy content only** (gallery pages, and any
Learning-article content describing a celestial object/event the same
way). If you're working on a K&E/slide-rule page instead, stop and use
`ke-stub-placeholder` — same box, different fields, different rules,
different footer links. Don't mix the two domains' conventions.

## Structure

```html
<div class="ke-ai-stub">
<span class="ke-ai-stub-badge">🤖 AI-drafted &middot; unverified</span>

<dl class="ke-ai-stub-facts">
<dt>What it is</dt>
<dd>...</dd>
<dt>Constellation</dt>
<dd>...</dd>
<dt>Distance</dt>
<dd>...</dd>
<dt>Apparent magnitude</dt>
<dd>...</dd>
<dt>Angular size</dt>
<dd>...</dd>
<dt>Coordinates</dt>
<dd>...</dd>
</dl>

<p class="ke-ai-stub-desc">This summary was generated by an AI assistant from general astronomical references, not from Jay's own notes on this specific image. Treat every detail above as a starting point for research, not settled fact.</p>

<div class="ke-ai-stub-footer">
<span class="tag">Verify further:</span> <a href="https://en.wikipedia.org/wiki/...">Wikipedia</a> &middot; <a href="http://www.messier.seds.org/m/m0XX.html">SEDS Messier Database</a>
</div>
</div>
```

- **`**bold**` markdown does not process inside this raw HTML block** —
  same gotcha as `.ke-scale-set` and every other raw-HTML component on
  this site. Use real `<strong>` if bolding is ever needed inside a
  field (rare — these are short factual fragments).
- **Fields adapt to the subject**, they are not a fixed schema. Star/
  cluster/nebula/galaxy pages use `What it is / Constellation / Distance
  / Apparent magnitude / Angular size / Coordinates` (see any of the ~50
  gallery entries built 2026-07-23 for the canonical shape). A comet or
  transient event uses whatever facts are actually relevant instead —
  e.g. the Comet NEOWISE page uses `What it is / Perihelion / Closest
  approach to Earth / Orbital period / Constellation (at the time)`, and
  the Great American Eclipse 2017 page uses `What it is / Path width /
  Maximum totality duration / Next similar U.S. event`. Always lead with
  `What it is`; after that, pick the 3-6 facts a curious reader would
  actually want for *that* kind of object, don't force-fit the star
  template onto a non-star subject.
- **The disclaimer paragraph's wording can vary slightly** by source
  (generic references vs. a specific cited site) but always states
  plainly that this is AI-compiled, not Jay's own notes on this specific
  subject, and that the reader should treat it as a research starting
  point, not settled fact. Don't soften or drop this — it's the entire
  point of the box.
- **Footer links** are astronomy references appropriate to the subject:
  Wikipedia is close to universal; add SEDS Messier Database
  (`http://www.messier.seds.org/m/m0XX.html`) for Messier objects.
  Verify any link resolves before hardcoding it — don't assume a URL
  pattern holds for a specific catalog number without checking.

## How to apply

1. Confirm this is genuinely AI-researched content, not something Jay
   wrote or dictated — if Jay supplies his own description of the same
   object, the stub box still gets added (per the 2026-07-23 rule
   change), it just sits alongside his prose rather than replacing it.
2. Research the real facts (type, distance, magnitude, angular size,
   coordinates, or event-specific equivalents) from general astronomical
   references — Wikipedia, SEDS, or similar. Don't invent plausible-
   sounding numbers.
3. Write the `<dl>` fields, the disclaimer paragraph, and a footer with
   working links.
4. Place the box once per page — typically at the end of the content,
   after any real Jay-written narrative, per the established pattern on
   every existing gallery entry.

## Verify

Build (`npx astro build`) and spot-check the rendered box in a preview —
confirm the badge, fact grid, italic disclaimer, and footer links all
render, and that no stray `**bold**` markdown leaked through unprocessed
(a tell that raw-HTML markdown parsing didn't happen the way you
expected).
